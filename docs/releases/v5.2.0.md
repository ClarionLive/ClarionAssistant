# Clarion Assistant 5.2

5.2 makes the CA Embeditor feel completely native — it now rides Clarion's **own** "Embeditor Source" command instead of a separate menu item, and appears with no flicker. It also adds a full **Code Snippets** workflow to the editor and moves the bundled language server to **pure upstream v0.9.8**.

**Supported:** Clarion 10, 11, and 12 (32‑bit IDE). The installer lets you pick which version(s) to install.

---

## Highlights

### 🩹 CA Embeditor rides Clarion's native "Embeditor Source"
There is no longer a separate **Open in CA Embeditor** popup item. Open an embed the way you always have — right‑click a procedure → **Embeditor Source**, or the **Embeditor** button on the Views toolbar — and the Monaco CA Embeditor overlays the native embeditor automatically. One embed at a time, exactly like the native editor; edits save straight back with **Save & Exit**.

- **Nothing new to learn** — the trigger is Clarion's existing command, so it works from everywhere that opens an embed.
- **Flicker‑free** — the overlay is placed the instant the embed view opens, before the native editor paints.
- **Toggle** it under **Options → Clarion Assistant → Editor Surfaces** ("Use the Monaco embeditor", on by default).
- The old right‑click menu injection and its native Windows‑hook machinery have been **retired** entirely.

*(Thanks to Mark Sarson — this adopts the auto‑detect approach from his [ClarionMonacoEditor](https://github.com/msarson/ClarionMonacoEditor).)*

### ✂️ Code Snippets in the editor
A full snippet workflow lands in the Monaco source editor and CA Embeditor:
- **Ctrl+J picker** — theme‑aware, with a **live expansion preview** so you see the resulting code (tab‑stops visible) before inserting.
- **Snippets in completion** — type a trigger and press **Enter** to expand (Clarion Ctrl+J parity), or use **trigger + space** for Clarion‑style expansion independent of completion.
- **Gear‑panel management** — edit snippets with examples/help, tab‑stop mirroring, and case‑insensitive extension scoping.

### 🧠 Pure upstream language server (v0.9.8)
The bundled LSP is now **stock upstream** [msarson/Clarion‑Extension](https://github.com/msarson/Clarion-Extension) **v0.9.8**. CodeGraph intelligence runs entirely on the CA (C#) side, so there is no patched server to drift out of date — future upstream releases are a clean one‑step re‑pin. A tag‑anchored sync mechanism keeps the pin honest.

## 🐞 Fixed

- **Read‑only files are honored** in the Monaco source editor — a `.clw`/`.inc` with the read‑only attribute now opens read‑only instead of letting you edit then failing silently at save time ([#50](https://github.com/ClarionLive/ClarionAssistant/issues/50)).
- **CodeGraph call relationships** now resolve `variable.Method()` callers and correctly attribute class‑method calls ([#54](https://github.com/ClarionLive/ClarionAssistant/issues/54)).
- **No UTF‑8 BOM on save** — the Monaco editor could write a BOM that Clarion rejected; writes are now BOM‑free CR/LF ([#34](https://github.com/ClarionLive/ClarionAssistant/issues/34)).
- **Snippet pipeline hardening** — mirror correctness, read‑only guard, save flush, and a dismissable Ctrl+J picker ([#49](https://github.com/ClarionLive/ClarionAssistant/issues/49) follow‑up).

## 📦 Install

1. Download **`ClarionAssistant-5.2-Setup.exe`** below (code‑signed).
2. Close the Clarion IDE.
3. Run the installer and pick your Clarion version(s) (10 / 11 / 12).
4. Start Clarion — the **Clarion Assistant** pane docks automatically (Tools → Clarion Assistant, or Ctrl+Alt+C).

Upgrading from 5.1 installs cleanly over the top.
