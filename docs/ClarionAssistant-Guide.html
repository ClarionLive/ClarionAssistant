Getting Started

Clarion.Assistant is an AI-powered addin that embeds Claude Code directly into your Clarion IDE, giving you an intelligent coding assistant that understands your entire codebase. This guide walks you through requirements, installation, and your first launch.

Overview

Clarion.Assistant bridges the gap between AI-assisted development and the Clarion IDE. Rather than switching between your editor and an external AI tool, the assistant lives inside your IDE with full awareness of your open files, solution structure, and class hierarchies.

Embedded AI Terminal

A Claude Code terminal panel docked directly in the IDE. It sees your active file, cursor position, and selected text — no copy-pasting context back and forth.

Real-Time Code Intelligence

Cross-file go-to-definition, find-all-references, and hover information powered by the Clarion Language Server. Ask the AI about any symbol and it can navigate you there instantly.

Class Helper

Analyze Clarion CLASS declarations in .inc files, compare them against .clw implementations, and generate missing method stubs automatically.

CodeGraph Indexing

Your entire solution is indexed into a SQL-queryable database. Ask questions like “who calls this procedure?” or “show me dead code” and get immediate, accurate answers.

Agent Monitoring

Monitor AI agent activity, review task progress, and coordinate work across multiple terminals through the MultiTerminal integration.

Task Lifecycle Board

A visual Kanban board for managing development tasks from creation through completion, integrated directly into the IDE workflow.

Installation

Clarion.Assistant is deployed as a single file — ClarionAssistant.dll — placed into the Clarion IDE's AddIns directory. No separate installer or configuration wizard is required.

1

Deploy the DLL

Copy ClarionAssistant.dll to your Clarion IDE's AddIns folder. The typical path is: 

C:\Program Files\SoftVelocity\ClarionXX\Bin\AddIns\ClarionAssistant.dll

Replace XX with your Clarion version number (e.g., Clarion11).

2

Launch the Clarion IDE

Start (or restart) the Clarion IDE. The addin is loaded automatically when the IDE initializes. You will see new menu entries under the Tools menu.

3

Automatic First-Run Configuration

On first launch, Clarion.Assistant performs the following setup automatically:

  • Clarion version detection — Scans for installed Clarion versions and selects the appropriate environment.
  • MCP configuration — Generates the Model Context Protocol configuration file at: 
    %APPDATA%\ClarionAssistant\mcp-config.json
    This file tells Claude Code how to communicate with the IDE tools.
  • Prompt file deployment — Creates a .claude/ folder in your working directory with a prompt file that configures Claude's behavior for Clarion development.
Note: The auto-generated configuration files are safe to inspect and customize. If you move your Clarion installation or upgrade to a new version, delete the MCP configuration file and restart the IDE to trigger re-detection.

First Launch

After installation, Clarion.Assistant adds three main panels to the IDE. Each can be opened from the Tools menu or with a keyboard shortcut.

Claude Chat

Ctrl + Alt + C

The primary AI assistant panel. An embedded Claude Code terminal with full awareness of your IDE state. It can read your active file, navigate to specific lines, make edits, analyze classes, and query your entire codebase.

Agent Panel

Ctrl + Alt + A

Monitors AI agent activity across your development environment. View active tasks, track progress, and coordinate with other terminals when using MultiTerminal.

Class Helper

Ctrl + Alt + M

A dedicated tool for working with Clarion CLASS structures. Analyze declarations in .inc files, compare against implementations in .clw files, and generate missing method stubs with a single click.

Your First Conversation

Once you have the Claude Chat panel open, try these common tasks to get familiar:

You type: "What file do I have open?"

The assistant reads your active editor tab and reports the file name, path, and a summary of its contents. This confirms that IDE integration is working.

You type: "Analyze the class in MyClass.inc"

The assistant parses the CLASS declaration, lists all methods and data members, and can check whether each method has a corresponding implementation in the .clw file.

You type: "Who calls the UpdateRecord procedure?"

Using CodeGraph, the assistant queries the indexed database and returns every call site across your entire solution, with file paths and line numbers.

You type: "Open Main.clw and go to line 150"

The assistant opens the file directly in your Clarion editor and positions the cursor at line 150.

Tip: The assistant is context-aware. If you select a block of code in the editor before asking a question, the assistant automatically sees your selection. Try selecting a procedure and asking "Explain this code" or "Find any bugs here".
Important: Clarion.Assistant requires an active Anthropic API key configured in your Claude Code CLI. If you see authentication errors in the chat panel, run claude auth in an external terminal to configure your credentials.

Claude Chat Terminal

The Claude Chat Terminal is the centerpiece of Clarion.Assistant. It embeds a fully functional Claude Code session directly inside the Clarion IDE, connected to over 30 MCP tools that give Claude real-time awareness of your editor, your solution, and your codebase. You can ask Claude to read your open file, navigate to a line, analyze a class hierarchy, or refactor code — all without leaving the IDE.

Solution Bar

The Solution Bar runs across the top of the Claude Chat panel. It controls which Clarion environment and solution Claude operates against.

Control Description
Version Selector Dropdown listing installed Clarion versions (11, 12, etc.). Clarion.Assistant auto-detects which versions are present on your system.
Solution Picker A dropdown showing up to 10 recently opened solutions, plus a browse button to locate a .sln file manually. The currently open solution in the IDE is auto-detected and pre-selected.
Refresh Forces re-detection of the IDE's currently open solution. Use this if you opened or switched solutions and the picker has not updated.
Full Index Triggers a complete CodeGraph indexing pass over the entire solution. This parses every source file and builds a .codegraph.db SQLite database containing all symbols, relationships, and call chains.

Index Status Displays the date and time of the last successful index. If the solution has never been indexed, this area prompts you to run a full index.
Tip: Run a Full Index once when you first open a solution, then use Update periodically as you make changes. Claude's CodeGraph queries are only as current as the last index.

The Terminal

The terminal area is a full terminal emulator powered by xterm.js. It renders inside a WebView2 control and connects to a real Windows ConPTY process running the Claude Code CLI.

Full Terminal Emulation

This is not a simplified chat window. It is a complete terminal that supports ANSI colors, cursor positioning, scrollback history, text selection, and copy/paste. Claude Code renders with full formatting — markdown, code blocks, and progress indicators.

IDE-Aware MCP Tools

Claude has direct access to the Clarion IDE through over 30 MCP tools. These tools let Claude interact with the editor, the file system, the application tree, the CodeGraph database, and the Language Server.

  • Get the selected text or the word under your cursor

  • Find all references to a symbol across the entire workspace

Class Helper

The Class Helper panel provides Clarion CLASS file analysis and code generation, bridging the gap between .inc declaration files and .clw implementation files. Access it via Ctrl+Alt+M or from the Tools menu.

Understanding Clarion Class Files

Clarion uses a paired-file convention for object-oriented code:

  • .inc files contain CLASS declarations — method prototypes and data member definitions that describe the class interface.
  • .clw files contain the actual method implementations, structured with MEMBER, INCLUDE, and MAP directives.

Class Helper understands both file types and automates the tedious work of keeping them in sync.

Panel Layout

The Class Helper window is divided into two resizable panes:

  • Class Tree (top) — A hierarchical tree view showing parsed class structures. Classes appear as top-level nodes, with methods and data members as children.
  • Output Pane (bottom) — A rich-text log displaying operation results, error messages, and generated code summaries.

A status bar along the bottom provides brief feedback on the current operation.

Toolbar Reference

Button Description When to Use
Analyze Parses the current .inc or .clw file for CLASS definitions. Populates the tree with class names, methods, and data members. When you want to explore or understand a class structure.
Generate Stubs Compares .inc declarations against .clw implementations and generates method stubs for any missing methods. After adding new method declarations to a .inc file.
Sync Check Bidirectional comparison reporting:
  • Missing methods — declared but not implemented
  • Orphaned methods — implemented but no longer declared
 For auditing class consistency and finding dead code.
New .clw Generates a complete .clw implementation file with proper MEMBER/INCLUDE/MAP structure. When starting a brand-new class.
Start MCP Toggles the Class Helper's embedded MCP server so Claude Code can invoke class tools directly. When using Claude Code alongside the IDE.

Tree Navigation

  • Double-click any node to jump directly to that line in the editor.
  • Class nodes display the class name and inheritance relationships.
  • Method nodes show the method name along with parameter signatures.
  • Data member nodes display the member name and its declared type.
Tip: After running Sync Check, the output pane lists missing and orphaned methods. Use this as a checklist to clean up your class files.

Typical Workflows

Starting a New Class

1

Write your CLASS declaration in a new .inc file.

2

Open the .inc file and click Analyze to verify the class tree.

3

Click New .clw to generate the complete implementation file.

4

Open the generated .clw file and fill in the method bodies.

Adding Methods to an Existing Class

1

Add new METHOD declarations to your .inc file.

2

With the .inc open, click Generate Stubs. Only missing methods are appended.

3

Open the .clw file and implement the newly generated stubs.

Auditing Class Consistency

1

Open either the .inc or .clw file and click Sync Check.

2

Review the output. Missing methods need implementations. Orphaned methods may be dead code to remove.

Tip: The Analyze button works on both .inc and .clw files. Analyzing the .inc shows the declared interface; analyzing the .clw shows what is actually implemented.

Agent Panel & Task Board

Requirement: These features require a running MultiTerminal instance (REST API on localhost:5050) for team collaboration and task management.

Agent Panel (Ctrl+Alt+A)

The Agent Panel is a dockable IDE pad that gives you a live view of every MultiTerminal agent connected to your workspace.

Live Agent List

Column Description
Agent Name The registered name of the MultiTerminal agent.
Active Task The title of the task the agent is working on, or (idle) if none.
Status Current agent state — active, idle, or offline.

Agents with an active task are rendered in black text; idle agents appear in gray.

Controls

  • Refresh — Forces an immediate update, bypassing the polling interval.
  • Open Board — Opens the Task Lifecycle Board for the selected agent's active task.
  • Double-click a row — Same as Open Board.

Auto-refreshes every 8 seconds (backs off to 15–30s on connection errors).

Task Lifecycle Board

A floating, kanban-style window for visual task management. Open it from the Agent Panel or the Task Board button in Claude Chat.

Board Layout

Four kanban columns representing the lifecycle phases:

Planning → Coding → Testing → Done
  • Checklist items appear as cards in their current column
  • Each column includes a phase notes section
  • Header shows: task title, description, priority, assignee, helpers
  • Footer shows: continuation notes for session handoff

Card Features

Drag-and-Drop

Cards follow a state machine: pending → coding → testing → done, with free back-and-forth between Coding and Testing.

Inline Editing

Click a card to edit its content directly on the board.

Agent Assignment

Individual cards can be assigned to a specific agent.

Cycle Count

Tracks how many times a card moves between Coding and Testing — a signal for items needing deeper investigation.

Transition Notes

Attach notes explaining why a card moved and what was tried, building a lightweight audit trail.

Real-Time Updates

  • Polling: Every 5 seconds, with exponential back-off on errors
  • Change detection: Lightweight hash comparison minimizes UI redraws
  • Minimized optimization: Polling paused when window is minimized
  • Immediate refresh: After drag-drop or inline edits

Multi-Board Support

  • Multiple boards can be open simultaneously, one per task
  • Opening a board for an already-open task reuses the existing window
  • Theme (dark/light) synchronized across all boards
  • Window position and size remembered between sessions

Language Server Integration

Clarion.Assistant includes integration with a Clarion Language Server Protocol (LSP) implementation, bringing modern code intelligence features to the Clarion IDE. The LSP provides real-time, compiler-grade analysis of your source code — enabling go-to-definition, find-references, hover documentation, and symbol search across your entire workspace.

Acknowledgment: The LSP integration in Clarion.Assistant was inspired by and builds upon the pioneering work of Mark Sarson, who developed LSP support for Clarion in his Clarion Extension for VS Code. Mark's extension demonstrated that modern language intelligence features — IntelliSense, go-to-definition, parameter hints, and more — could be brought to Clarion development. His work laid the groundwork that made this integration possible. Special thanks to Mark for his contributions to the Clarion developer community.

Capabilities

Go to Definition

Jump directly to where any symbol is defined, even across files and projects. Select a procedure name, variable, or class reference and Claude can navigate you to its declaration instantly using lsp_definition.

Find All References

Locate every usage of a symbol across your entire workspace. Ask Claude “where is this used?” and it queries lsp_references to return every call site, assignment, and reference with file paths and line numbers.

Hover Information

Get type information, method signatures, and documentation for any symbol. Claude uses lsp_hover to provide detailed context about variables, procedures, and class members without you needing to find the declaration yourself.

Symbol Search

Search for symbols by name across the entire workspace with lsp_find_symbol, or list all symbols in a specific file with lsp_document_symbols. Useful for quickly locating procedures, classes, or variables when you know part of the name.

How It Works

The Language Server starts automatically when you select a solution in the Claude Chat panel. It runs in the background, analyzing your Clarion source files and maintaining an up-to-date model of your codebase.

LSP vs. CodeGraph

Clarion.Assistant offers two complementary code intelligence systems:

Feature LSP CodeGraph
Analysis type Real-time, live analysis of current source Indexed snapshot of the codebase
Best for Precise definitions, references, type info Call chains, impact analysis, dead code detection
Scope Current workspace Entire solution (SQL-queryable)
Update frequency Continuous (as you edit) On-demand (Full Index / Update)

Claude uses both systems and chooses the best one for each query automatically. For simple “where is X defined?” questions, the LSP is preferred. For complex cross-project queries like “what breaks if I change X?”, CodeGraph is used.

Example Queries

You Ask LSP Tool Used
“Where is this procedure defined?” lsp_definition — jumps to the declaration
“Who uses this variable?” lsp_references — lists all usage sites
“What type is this?” lsp_hover — shows type and signature info
“What procedures are in this file?” lsp_document_symbols — lists all symbols
“Find a procedure called UpdateCustomer” lsp_find_symbol — searches by name

You do not need to call these tools yourself — Claude selects and invokes them automatically based on your questions and requests. This reference is provided so you know what capabilities are available.

CodeGraph (Solution-Wide Intelligence)

Indexed, queryable access to every symbol, relationship, and call chain across your entire solution.

ToolDescription
query_codegraphRun SQL queries against the indexed CodeGraph database to find symbols, trace call chains, and analyze dependencies.
list_codegraph_databasesFind available indexed CodeGraph databases.
CodeGraph Database Schema

The CodeGraph database contains three main tables:

  • symbols — name, type (procedure, function, class, interface, routine, variable), file_path, line_number, params, return_type, parent_name, scope
  • relationships — from_id, to_id, type (calls, do, inherits, implements, references), file_path, line_number
  • projects — name, cwproj_path, sln_path

Example questions: “Who calls this procedure?”, “Show me dead code”, “What is the class hierarchy?”, “If I change X, what breaks?”
